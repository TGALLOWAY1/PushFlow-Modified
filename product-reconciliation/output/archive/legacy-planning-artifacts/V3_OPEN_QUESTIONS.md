# V3 Open Questions

Only the decisions that still require human judgment are listed here.

1. Should the primary user-facing container be named `Project` or
   `Song Project`? Reason: `Project` is cleaner for the workflow, but the
   imported material is often song-based and the entry experience may still want
   that language.

2. Should `Natural Hand Pose` be required early in onboarding, or introduced
   only when the user first generates or optimizes a layout? Reason: this
   changes the first-run friction and the degree to which personalization is
   part of the core promise.

3. Is timeline editing part of V3's core workflow, or should the timeline be
   inspection-first in the first V3 release? Reason: this determines whether V3
   is primarily a verification tool with editing support or a full
   performance-authoring environment.

4. Is Push/Ableton export part of the first V3 product definition, or a post-V3
   capability? Reason: if export is core, V3 becomes a downstream production
   tool; if not, it remains a focused analysis-and-layout tool.

5. Should pattern generation remain completely out of V3, or be deferred as a
   clearly separate adjunct workflow? Reason: keeping it in the main shell risks
   reintroducing the same mission drift that weakened V2.
